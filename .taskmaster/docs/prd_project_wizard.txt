# Product Requirements Document: Project Wizard & Task Management

## Project: GameSmith Project Wizard with AI-Powered PRD Generation
**Version:** 1.0
**Date:** 2025-10-09
**Author:** Unity GameSmith Team

---

## 1. Executive Summary

Create a modern, slick UI wizard that guides developers through project initialization, from game concept to AI-generated PRD to tracked task list. This wizard integrates with Taskmaster MCP to provide end-to-end project planning and task management within Unity Editor.

---

## 2. Problem Statement

**Current Situation:**
- Developers start Unity projects without structured planning
- No systematic way to capture game requirements and technical decisions
- PRD creation is manual and time-consuming
- No built-in task tracking or project status visibility
- Tech stack decisions (UniTask, DOTween, Mirror, Netcode) are made ad-hoc

**Desired Outcome:**
- One-click project initialization with guided wizard
- AI generates MVP PRD based on user inputs
- Automatic task list creation from PRD
- Visual task dashboard showing project status
- Structured tech stack selection and package management
- Seamless workflow: Concept â†’ PRD â†’ Tasks â†’ Implementation

---

## 3. Goals & Success Metrics

**Primary Goals:**
1. Launch wizard that captures game concept in 5 minutes or less
2. Generate publication-ready PRD with AI assistance
3. Automatically create task list from PRD using Taskmaster MCP
4. Provide real-time visual task status dashboard
5. Streamline tech stack selection and package installation

**Success Metrics:**
- Wizard completion rate: >80%
- PRD generation time: <30 seconds
- Task list accuracy: >90% (tasks match PRD requirements)
- User adoption: 70%+ of new projects use wizard
- Time to first task: <10 minutes from wizard start
- User satisfaction: 4.5+ stars for wizard experience

---

## 4. User Stories

**As a** Unity game developer
**I want** a step-by-step wizard to initialize my project
**So that** I can structure my game development with AI-generated planning and task tracking

**Acceptance Criteria:**
- [ ] Wizard opens with modern, intuitive UI
- [ ] Step 1: Capture game basics (genre, multiplayer, theme)
- [ ] Step 2: Select tech stack (UniTask, DOTween, networking, etc.)
- [ ] Step 3: AI generates MVP PRD based on inputs
- [ ] Step 4: User can edit PRD before submission
- [ ] Step 5: Taskmaster parses PRD into task list
- [ ] Step 6: Visual dashboard shows task status
- [ ] Separate "Project Wizard" menu item in Tools
- [ ] Can return to wizard to update project details

## 5. Technical Requirements

### 5.1 Unity Integration
- Unity Version: 2021.3 LTS+
- Platform Support: Editor-only tool
- Dependencies:
  - Existing GameSmith AI system
  - Taskmaster MCP (already integrated)
  - Unity Package Manager (for tech stack installation)
  - EditorWindow with multi-step wizard UI

### 5.2 Code Generation
- Template Type: EditorWindow + ScriptableObject data model
- Namespace: SparkGames.UnityGameSmith.Editor.Wizard
- Script Location: Assets/Scripts/Editor/Wizard/
- Data Location: ProjectSettings/GameSmithProjectData.asset

### 5.3 AI Provider Compatibility
- [x] Works with Ollama (local)
- [x] Works with Claude (cloud) - Recommended for PRD generation
- [x] Works with OpenAI (cloud)
- [x] Works with custom endpoints

---

## 6. Feature Specification

### 6.1 Core Functionality

**Key Components:**

1. **ProjectWizardWindow**
   - Purpose: Multi-step wizard EditorWindow with modern UI
   - Implementation: Step-based navigation with progress indicator
   - Dependencies: AIAgentClient, TaskmasterMCPClient, GameSmithWindow

2. **ProjectConfigData**
   - Purpose: ScriptableObject storing project configuration
   - Implementation: Persistent storage of wizard inputs
   - Dependencies: None (pure data class)

3. **TechStackManager**
   - Purpose: Manages Unity package installation and configuration
   - Implementation: Unity Package Manager API wrapper
   - Dependencies: UnityEditor.PackageManager

4. **PRDGenerator**
   - Purpose: AI-powered PRD generation from wizard inputs
   - Implementation: Structured prompt engineering with Claude/GPT
   - Dependencies: AIAgentClient

5. **PRDEditorWindow**
   - Purpose: Rich text editor for PRD modification
   - Implementation: Multi-line TextArea with syntax highlighting
   - Dependencies: None

6. **TaskDashboardWindow**
   - Purpose: Visual task status dashboard with progress tracking
   - Implementation: EditorWindow with task list UI, progress bars, filters
   - Dependencies: TaskmasterMCPClient

7. **TaskmasterMCPClient**
   - Purpose: Interface to Taskmaster MCP for PRD parsing and task management
   - Implementation: Wrapper around Taskmaster MCP API
   - Dependencies: .mcp.json configuration

## 11. Risks & Mitigations

|------|--------|-------------|------------|
| AI PRD generation quality varies | High | Medium | Provide structured prompts, fallback templates, user editing |
| Package installation failures | High | Medium | Error handling, retry logic, manual installation option |
| Taskmaster MCP parsing errors | High | Low | Validate PRD format before parsing, error feedback |
| Wizard too complex for beginners | Medium | Medium | Progressive disclosure, tooltips, skip options |
| Performance with large task lists | Medium | Low | Pagination, lazy loading, filtering |
| Unity version compatibility | Low | Low | Test with 2021.3 LTS baseline, graceful degradation |
